# WhereNow Activity Diagram 分析

## 一、Activity Diagram 說明
Activity Diagram（活動圖）用來描述系統中某項功能的執行流程，呈現使用者與系統在操作過程中的步驟順序與邏輯關係。透過活動圖，可以更清楚看出功能從開始到結束的完整流程。

本文件以「使用者建立回憶流程」作為分析主題，說明使用者如何在系統中新增地點並建立一筆回憶紀錄。

---

## 二、流程主題
**使用者建立回憶流程**

---

## 三、流程步驟說明
1. 使用者登入系統
2. 進入地點管理或回憶新增頁面
3. 判斷是否已有對應地點資料
4. 若無地點資料，先新增地點
5. 使用者進入新增回憶頁面
6. 輸入回憶標題、內容、日期等資訊
7. 上傳照片
8. 系統儲存回憶資料
9. 顯示新增成功訊息
10. 流程結束

---

## 四、流程邏輯整理
- 若使用者尚未建立地點，需先完成地點新增
- 地點建立完成後，才能將回憶與地點關聯
- 回憶資料輸入完成並儲存後，系統回傳成功結果

---

## 五、Activity Diagram 繪製建議
圖中應包含以下幾種元素：

- **開始節點**：黑色實心圓
- **活動節點**：長方形圓角框
- **判斷節點**：菱形
- **箭頭**：表示流程方向
- **結束節點**：雙圓結束符號

建議流程如下：

開始  
→ 登入系統  
→ 進入地點頁面  
→ 判斷是否已有地點  
→（否）新增地點  
→ 進入新增回憶頁面  
→ 輸入回憶內容  
→ 上傳照片  
→ 儲存回憶  
→ 顯示成功訊息  
→ 結束
